## 什么是Dubbo

Dubbo是一款高性能、轻量级的开源Java RPC框架，致力于提供高性能和透明化的RPC远程服务调用方案，以及SOA服务治理方案。

### Dubbo核心特性

- **面向接口代理**：像调用本地服务一样调用远程服务
- **智能负载均衡**：支持多种负载均衡策略
- **自动服务注册和发现**：支持服务自动注册和发现
- **高度可扩展**：协议、传输、序列化都可扩展
- **运行期流量控制**：支持限流、降级、超时等控制
- **灰度发布**：支持灰度发布，多版本并行
- **服务治理**：支持服务降级、熔断、监控等

## Dubbo架构

### 整体架构

```
┌──────────┐
│ Provider │ (服务提供者)
└─────┬────┘
      │ register
      v
┌──────────────────┐
│ Registry Server  │ (注册中心)
└─────┬────────────┘
      │ subscribe
      v
┌──────────┐
│ Consumer │ (服务消费者)
└──────────┘
```

### 5层架构

1. **业务层(Business)**：业务逻辑实现
2. **接口层(Interface)**：服务接口定义
3. **协议层(Protocol)**：RPC协议实现
4. **传输层(Transport)**：网络传输
5. **网络层(Network)**：物理网络通信

## Dubbo快速开始

### 1. 定义服务接口

```java
// DemoService.java (service-api模块)
public interface DemoService {
    String sayHello(String name);
}
```

### 2. 实现服务提供者

#### 2.1 依赖配置

```xml
<dependency>
    <groupId>org.apache.dubbo</groupId>
    <artifactId>dubbo</artifactId>
    <version>3.0.0</version>
</dependency>

<dependency>
    <groupId>org.apache.curator</groupId>
    <artifactId>curator-framework</artifactId>
    <version>4.13.0</version>
</dependency>

<dependency>
    <groupId>org.apache.curator</groupId>
    <artifactId>curator-recipes</artifactId>
    <version>4.13.0</version>
</dependency>
```

#### 2.2 实现服务

```java
// DemoServiceImpl.java
@DubboService
public class DemoServiceImpl implements DemoService {
    @Override
    public String sayHello(String name) {
        return "Hello " + name;
    }
}
```

#### 2.3 配置应用

```properties
# application.properties
dubbo.application.name=demo-provider
dubbo.registry.address=zookeeper://127.0.0.1:2181
dubbo.protocol.name=dubbo
dubbo.protocol.port=20880
```

或使用XML配置：

```xml
<!-- dubbo-provider.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:dubbo="http://dubbo.apache.org/schema/dubbo"
   xsi:schemaLocation="http://www.springframework.org/schema/beans 
   http://www.springframework.org/schema/beans/spring-beans-4.3.xsd
   http://dubbo.apache.org/schema/dubbo 
   http://dubbo.apache.org/schema/dubbo/dubbo.xsd">

    <!-- 应用名称 -->
    <dubbo:application name="demo-provider"/>
    
    <!-- 注册中心 -->
    <dubbo:registry address="zookeeper://127.0.0.1:2181"/>
    
    <!-- RPC协议 -->
    <dubbo:protocol name="dubbo" port="20880"/>
    
    <!-- 实现类 -->
    <bean id="demoService" class="com.example.DemoServiceImpl"/>
    
    <!-- 服务暴露 -->
    <dubbo:service interface="com.example.DemoService" ref="demoService"/>
</beans>
```

### 3. 实现服务消费者

#### 3.1 配置应用

```properties
# application.properties
dubbo.application.name=demo-consumer
dubbo.registry.address=zookeeper://127.0.0.1:2181
```

#### 3.2 调用服务

```java
// Main.java
@SpringBootApplication
public class ConsumerApplication {
    
    @DubboReference
    private DemoService demoService;
    
    public static void main(String[] args) {
        SpringApplication.run(ConsumerApplication.class, args);
    }
    
    @PostConstruct
    public void test() {
        String result = demoService.sayHello("World");
        System.out.println(result);
    }
}
```

或使用XML配置：

```xml
<!-- dubbo-consumer.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:dubbo="http://dubbo.apache.org/schema/dubbo"
   xsi:schemaLocation="http://www.springframework.org/schema/beans 
   http://www.springframework.org/schema/beans/spring-beans-4.3.xsd
   http://dubbo.apache.org/schema/dubbo 
   http://dubbo.apache.org/schema/dubbo/dubbo.xsd">

    <!-- 应用名称 -->
    <dubbo:application name="demo-consumer"/>
    
    <!-- 注册中心 -->
    <dubbo:registry address="zookeeper://127.0.0.1:2181"/>
    
    <!-- 服务引用 -->
    <dubbo:reference id="demoService" interface="com.example.DemoService"/>
</beans>
```

## 常见配置详解

### Provider配置

| 配置项 | 说明 | 默认值 |
|------|------|--------|
| `timeout` | 服务调用超时时间(毫秒) | 1000 |
| `retries` | 服务调用失败重试次数 | 2 |
| `loadbalance` | 负载均衡策略 | random |
| `cluster` | 集群方式 | failover |
| `version` | 服务版本 | 0.0.0 |

### Consumer配置

| 配置项 | 说明 | 默认值 |
|------|------|--------|
| `timeout` | 服务调用超时时间(毫秒) | 1000 |
| `retries` | 服务调用失败重试次数 | 2 |
| `loadbalance` | 负载均衡策略 | random |
| `check` | 启动时检查提供者是否存在 | true |
| `version` | 调用服务版本 | 0.0.0 |

## 负载均衡策略

Dubbo提供多种负载均衡策略：

1. **Random Load Balance** (随机)：按权重随机选择
2. **RoundRobin Load Balance** (轮询)：按权重轮询
3. **LeastActive Load Balance** (最少活跃调用)：调用活跃数越少越优先
4. **ConsistentHash Load Balance** (一致性哈希)：同一消费者的调用总是选中同一提供者

配置方式：

```properties
dubbo.consumer.loadbalance=roundrobin
```

## 集群容错

Dubbo支持多种容错模式：

| 容错模式 | 说明 |
|---------|------|
| Failover | 失败自动切换，建议用于幂等操作 |
| Failfast | 快速失败，建议用于非幂等操作 |
| Failsafe | 失败安全，忽略调用异常 |
| Failback | 失败自动恢复，背后发起重试 |
| Forking | 并行调用多个提供者 |
| Broadcast | 广播调用所有提供者 |

配置方式：

```properties
dubbo.consumer.cluster=failover
```

## 服务降级

当服务故障时进行降级处理：

```java
@Reference(version = "1.0.0", cluster = "failsafe", mock = "com.example.DemoServiceMock")
private DemoService demoService;
```

或配置方式：

```xml
<dubbo:reference id="demoService" 
    interface="com.example.DemoService"
    cluster="failsafe"
    mock="com.example.DemoServiceMock"/>
```

## 服务版本

实现多个版本的同一接口：

提供者：
```xml
<dubbo:service interface="com.example.DemoService" 
    ref="demoService" 
    version="1.0.0"/>
<dubbo:service interface="com.example.DemoService" 
    ref="demoServiceV2" 
    version="2.0.0"/>
```

消费者：
```xml
<dubbo:reference id="demoService" 
    interface="com.example.DemoService" 
    version="1.0.0"/>
```

## 协议支持

Dubbo支持多种协议：

- **Dubbo协议**：默认，基于 Netty 的高性能协议
- **HTTP协议**：基于 HTTP 的协议
- **Thrift协议**：跨语言协议
- **REST协议**：基于 HTTP REST 的协议

## 常见问题解决

### 1. 启动报错找不到服务

检查：
- 是否启动了注册中心（Zookeeper）
- 提供者是否正确注册
- 消费者配置的接口是否正确

### 2. 调用超时

解决方案：
- 增加超时时间
- 优化业务逻辑性能
- 检查网络连接

### 3. 服务找不到

检查：
- 注册中心是否正常
- 服务版本是否匹配
- 接口类路径是否一致
